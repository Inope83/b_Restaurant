# 🤝 Contributing to b_Restaurant

Thank you for your interest in contributing! Here's how to get started.

---

## 🚀 Getting Started

1. **Fork** this repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/your-username/b_Restaurant.git
   cd b_Restaurant
   ```
3. **Create a branch** for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Set up** the project (see README.md)
5. **Make your changes**
6. **Commit** with a clear message:
   ```bash
   git commit -m "feat: add table rating feature"
   ```
7. **Push** and open a **Pull Request**

---

## 📌 Commit Message Format

Use this format for commit messages:

| Prefix | When to use |
|---|---|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `docs:` | Documentation changes |
| `style:` | Formatting, no logic change |
| `refactor:` | Code restructure |
| `test:` | Adding tests |

---

## ✅ Pull Request Checklist

- [ ] Code follows existing style
- [ ] Migrations included if models changed
- [ ] README updated if needed
- [ ] No `db.sqlite3` or `.env` committed
